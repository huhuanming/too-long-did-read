# Claude 现已在 Microsoft Foundry 和 Microsoft 365 Copilot 中可用
# Claude Now Available in Microsoft Foundry and Microsoft 365 Copilot

> 原文链接: https://www.anthropic.com/news/claude-in-microsoft-foundry
> 日期: 2025-11-18

---

## English

Today we announced that Microsoft and Anthropic are expanding our partnership. Claude Sonnet 4.5, Haiku 4.5, and Opus 4.1 models are now available in public preview in Microsoft Foundry, where Azure customers can build production applications and enterprise agents.

This enables companies to build with Claude, the world's best models for coding, agents, and office tasks, all while using their existing Microsoft ecosystem. Developers can also use Claude models in Microsoft Foundry with Claude Code.

In addition to existing integrations with Microsoft 365 Copilot—where Claude powers the Researcher agent for complex, multistep research, and enables custom agent development in Copilot Studio—Microsoft's Agent Mode in Excel now includes an option to use Claude in preview to build and edit spreadsheets directly in Excel.

### Build with Claude in Microsoft Foundry

Claude is available in Microsoft Foundry via serverless deployment. This integration enables developers to:

- **Start building immediately:** Deploy Claude through Foundry's APIs, tools, and workflows
- **Use existing Azure agreements:** Claude is eligible for Microsoft Azure Consumption Commitment (MACC), eliminating separate vendor approvals
- **Build in preferred language:** Access Claude using Python, TypeScript, and C# SDKs with Microsoft Entra authentication

### Select the right model for your use case

**Sonnet 4.5** is the best coding model in the world and the strongest model for building complex agents.

**Haiku 4.5** is the fastest model delivering near-frontier performance at one-third the cost of Sonnet.

**Opus 4.1** is an exceptional model for specialized reasoning tasks.

All models support Claude Developer Platform capabilities in Foundry, including code execution, web search and fetch, citations, vision, tool use, prompt caching, and more.

## 中文翻译

今天我们宣布微软和 Anthropic 正在扩大合作伙伴关系。Claude Sonnet 4.5、Haiku 4.5 和 Opus 4.1 模型现在在 Microsoft Foundry 中以公开预览形式提供，Azure 客户可以在此构建生产应用和企业代理。

这使公司能够使用 Claude 进行构建——世界上最好的编程、代理和办公任务模型——同时使用他们现有的微软生态系统。开发者还可以在 Microsoft Foundry 中使用 Claude Code。

除了与 Microsoft 365 Copilot 的现有集成——Claude 为研究员代理提供复杂多步骤研究支持，并在 Copilot Studio 中实现自定义代理开发——微软 Excel 中的代理模式现在包括一个选项，可以在预览中使用 Claude 直接在 Excel 中构建和编辑电子表格。

### 在 Microsoft Foundry 中使用 Claude 构建

Claude 通过无服务器部署在 Microsoft Foundry 中提供。此集成使开发者能够：

- **立即开始构建：** 通过 Foundry 的 API、工具和工作流部署 Claude
- **使用现有 Azure 协议：** Claude 符合 Microsoft Azure 消费承诺 (MACC) 资格，无需单独的供应商审批
- **使用首选语言构建：** 使用 Python、TypeScript 和 C# SDK 通过 Microsoft Entra 认证访问 Claude

### 为您的用例选择合适的模型

**Sonnet 4.5** 是世界上最好的编程模型，也是构建复杂代理的最强模型。

**Haiku 4.5** 是最快的模型，以 Sonnet 三分之一的成本提供接近前沿的性能。

**Opus 4.1** 是专门推理任务的出色模型。
